## ONE open item — curl (design-call, needs chakrit's yes/no)
**curl** has no flag to key on: the body is whatever the server returns, and the filter
caps it ("passthrough body, cap only"). A `curl -s api | jq` of a multi-line JSON body IS
truncated today. Proposed one-liner fix (logged in the audit note): sniff the first body
line after the headers — starts with `{`/`[` → raw the whole body; else cap as today. Is
that heuristic wanted, or is body-capping accepted? Not fixed unilaterally.
